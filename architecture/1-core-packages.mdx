Sage’s core architecture is built around modular smart contract packages, each designed for specific tasks but seamlessly integrated to power the entire protocol. This makes it simple for developers to mix, match, and upgrade components—enabling quick customization, easier integration, and accelerated innovation.

<Card title="Admin Package">
  Oversees administrative capabilities, including configurable protocol parameters, maintaining the protocol’s flexibility and responsiveness.
</Card>

<Card title="Analytics Package">
  Tracks user and content engagement metrics, providing valuable insights to inform interactions and reward mechanisms.
</Card>

<Card title="Channel Package">
  Facilitates the creation, management, and moderation of social channels, enabling structured and meaningful interactions within communities

  <Info>
    
  </Info>
</Card>

<Card title="Post Package">
  Enables decentralized content posting, commenting, and content engagement tracking, providing a comprehensive content-management framework.
</Card>

<Card title="Rewards Package">
  Implements the Proof of Social Contribution (PoSC) validation mechanism, systematically determining the eligibility and distribution of rewards tied explicitly to validated social actions.
</Card>

<Card title="Shared Package">
  Defines the structs and functionality for data types that are used in the other packages (e.g. managing friends, follows, likes, and posts, etc.).
</Card>

<Card title="Trust Package">
  Defines the \$TRUST coin and methods for minting and burning according to the logic defined in the Rewards package.
</Card>

<Card title="User Package">
  Manages decentralized user identities, profile creation, authentication, user-specific actions, and relationships.

  <Info>
    **Key Package Interactions**:

    - User onboarding
    - Identity creation
    - User-specific actions
  </Info>
</Card>

<Card title="Utils Package">
  Contains simple operations to act on basic types that are used consistently (e.g. string concatenation).
</Card>
